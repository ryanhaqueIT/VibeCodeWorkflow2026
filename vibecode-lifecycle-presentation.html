Lifecycle

15-Step Workflow for High-Context AI Coding

Disciplined · Reproducible · Automated

The Problem

AI coding without discipline leads to drift, hallucinations, and unmaintainable code.

  • Vague prompts cause inconsistent outputs
  • No shared understanding = rework and frustration
  • AI "sounds right" even when wrong
  • Context gets lost across sessions
  • No audit trail for AI-generated changes

"The VibeCode Lifecycle brings structure to chaos."

The 15-Step Workflow

Four phases, one disciplined process.

Discovery

Steps 0-2

Define problem, clarify requirements, write spec.md

Planning

Steps 3-5

Generate plan.md, critique & refine, set guardrails

Execution

Steps 6-11

Small chunks, context pack, implement, test, debug

Review

Steps 12-15

Human review, second opinion, commit, iterate

Phase 1

Discovery Steps 0-2

0

Problem Statement

Define boundaries in 1-3 sentences. Prevents vague prompting that causes drift.

1

Outcome Discovery

"Ask me questions until requirements + edge cases are clear." Forces shared understanding.

2

Write spec.md

Consolidate into spec.md: requirements, architecture, data model, test strategy.

Output: spec.md — your stable source of truth

Phase 2

Planning Steps 3-5

3

Generate plan.md

Break work into small tasks with milestones + definition of done for each.

4

Critique & Refine

Iteratively improve plan for gaps, risks, sequencing. Add test checklist per step.

5

Set Guardrails

Create rules.md: coding conventions, forbidden patterns, "if unsure ask".

Why: Fix issues before code exists—cheaper than refactoring later

Phase 3

Execution Loop
Steps 6-11

Small chunks, focused prompts, tight feedback loops.

6 → 7 → 8 → 9 → 10 → 11 ↺

Select task → Pack context → Implement → Test → Debug → Repeat

Step 7

Context Packing

The secret sauce. Provide relevant context for THIS task only.

What to Include

  • Files to modify/reference
  • Invariants & non-goals
  • Relevant docs / API snippets
  • Pitfalls to avoid

Tools

  • gitingest — Repo bundling
  • Context7 MCP — Live API docs
  • Serena — Symbolic navigation
  • Chrome DevTools — UI context

Output: context-pack.md — focused context saves tokens, improves accuracy

Steps 8-11

Implement → Test → Debug

8

Implement Step N

Prompt includes: spec.md refs + plan step + rules.md + context-pack. Small diff only.

9

Run Tests

Unit tests, lint, typecheck, build. Tests are the safety net—AI "sounds right" even when wrong.

10

Green?

If YES → Human review. If NO → Debug loop with logs and traces.

11

Debug Loop

Paste failures + traces; AI proposes fix; re-run. AI excels at iterate→test→fix.

Phase 4

Review & Commit

12

Human Review

Can you explain it? Is it simple/maintainable? Accountability gate.

13

Second Model Review

Different model critiques diffs. "Second opinion" catches what first model missed.

14

Commit

Small commit, clear message. Never commit what you can't explain. Audit trail.

15

Next Step?

More tasks in plan.md? Loop to Step 6. Done? PR/Merge with CI gates.

Key Takeaways

  • Discipline beats speed — Upfront planning prevents costly rework
  • Small chunks, tight loops — LLMs excel on focused prompts
  • Context is king — Pack relevant context per task, not entire repos
  • Tests are safety nets — AI sounds confident even when wrong
  • Human accountability — Never commit what you can't explain
  • Automate the boring — Scripts and skills handle repetitive steps

"Structure your AI workflow before your AI structures your codebase."
